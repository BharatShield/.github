# 🛡️ Bharat Shield V2

![Bharat Shield Logo](https://raw.githubusercontent.com/BharatShield/.github/main/profile/BharatShield_640.png)

**A Unified Command & Control Framework for National Digital Infrastructure**

## 🎯 The Mission
To provide high-end IT Services at a nominal cost for Education, Healthcare, and Public Distribution Systems through a sovereign digital nervous system. Bharat Shield acts as a "Sovereign Overlay," connecting existing fragmented systems to provide unified identity, data, and security management.

## 🏗️ Core Repositories
This organization is divided into four primary, interdependent modules:

*   **[`uem`](https://github.com/BharatShield/uem):** Unified Endpoint Management (MDM/EMM). Securely manages endpoints, devices, and access.
*   **[`itsm`](https://github.com/BharatShield/itsm):** Information Technology Service Management. Handles monitoring, logging, predictive analytics, and workflow management.
*   **[`siem`](https://github.com/BharatShield/siem):** Security Information and Event Management. Real-time threat detection, log aggregation, and active defense.
*   **[`mogoj`](https://github.com/BharatShield/mogoj):** Modular Orchestration of Gigantic Operational Judgment. The AI Core driving predictive insights and automated remediation.

## 🛠️ Technology Stack & Engineering Standards
To maintain high performance, security, and developer velocity, all code generated within this organization MUST adhere to the following standards:

*   **Backend Services:** Python-based using **FastAPI** for high-performance, asynchronous REST/GraphQL APIs.
*   **Unified Clients:** Built with **Flutter** to ensure seamless cross-platform functionality across all operational dashboards.
*   **Infrastructure & Deployment:** All services must be fully containerized using **Docker** and orchestrated behind **Nginx** reverse proxies. 
*   **Observability:** Native integration of the **OpenTelemetry Protocol (OTLP)** for distributed tracing, metrics, and logging across all microservices is strictly required. 
*   **Documentation:** We follow a strict **Docs-as-Code** philosophy. All repository documentation must be structured for seamless rendering via **Mintlify**.
